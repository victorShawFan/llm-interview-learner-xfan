# 大模型压缩技术

## 为什么需要模型压缩？

大型语言模型（如GPT-3、LLaMA、ChatGLM）参数量动辄数十亿甚至千亿，带来：
- 存储成本高：175B模型需要700GB（FP16）
- 推理延迟大：单次前向传播耗时数秒
- 部署困难：无法在消费级GPU或移动设备运行

模型压缩技术通过减小模型体积、加速推理，同时尽量保持性能。

## 主流压缩技术

### 1. 量化（Quantization）

将模型权重从高精度（FP32/FP16）降低到低精度（INT8/INT4）

**技术细节**：
- PTQ（Post-Training Quantization）：训练后量化，无需重训练
  - 将权重W从FP16映射到INT8：`Q(W) = round(W / scale) + zero_point`
  - scale和zero_point从训练集统计得出
  
- QAT（Quantization-Aware Training）：训练时模拟量化过程
  - 前向传播使用量化权重，反向传播使用FP32梯度
  - 模型学会适应量化误差

**实际应用**：
- GPTQ：针对GPT类模型的4-bit量化方案，精度损失<1%
- LLM.int8()：混合精度方案，敏感层用FP16，其余用INT8
- AWQ（Activation-aware Weight Quantization）：根据激活值重要性分配量化位宽

**效果**：
- INT8量化：模型大小减半，推理速度提升1.5-2x
- INT4量化：模型大小降至1/4，推理速度提升2-3x

### 2. 剪枝（Pruning）

移除模型中不重要的参数或结构

**剪枝类型**：
- 非结构化剪枝：移除单个权重（不规则稀疏，需专用硬件）
  - 基于L1范数：移除绝对值最小的权重
  - 基于梯度：移除梯度小的权重
  
- 结构化剪枝：移除整个神经元、注意力头、层（规则稀疏，通用硬件友好）
  - 移除低重要性的注意力头
  - 移除冗余的MLP中间层神经元

**LLM剪枝实践**：
- LLM-Pruner：自动识别并剪除Transformer中冗余结构
- SparseGPT：一次性剪枝算法，无需微调即可达到50%稀疏度

**效果**：
- 20-30%稀疏度：性能几乎无损
- 50%稀疏度：精度下降2-5%，推理速度提升40-50%

### 3. 知识蒸馏（Knowledge Distillation）

用大模型（Teacher）指导小模型（Student）学习

**蒸馏目标**：
- 软标签蒸馏：Student模仿Teacher的输出分布
  - `Loss = KL(P_student || P_teacher) + CE(P_student, y_true)`
  
- 特征蒸馏：Student模仿Teacher的中间层表示
  - `Loss_hidden = MSE(H_student, H_teacher)`

**LLM蒸馏实践**：
- TinyBERT：12层BERT蒸馏为4层，保留96.8%性能
- DistilGPT：GPT-2蒸馏为一半大小，困惑度仅增加2%
- MiniLLM：通过逆KL散度蒸馏，避免Student过度自信

**效果**：
- 模型参数减少50-70%
- 推理速度提升2-3x
- 精度保持90-95%

### 4. 低秩分解（Low-Rank Decomposition）

将权重矩阵分解为低秩矩阵乘积

**技术原理**：
- 奇异值分解（SVD）：`W ∈ R^(m×n) ≈ U·Σ·V^T`
  - 保留前k个奇异值，压缩比：`k(m+n) / (m×n)`
  
- LoRA（Low-Rank Adaptation）：
  - 冻结预训练权重W，添加低秩更新：`W' = W + A·B`
  - A ∈ R^(m×r), B ∈ R^(r×n)，其中r << min(m,n)

**应用场景**：
- LoRA微调：仅训练0.1%参数，效果接近全量微调
- TensorGPT：对Transformer的QKV矩阵进行低秩分解

**效果**：
- 参数量减少10-50%（取决于秩r）
- 推理速度提升10-30%

## 压缩技术对比

| 技术 | 压缩比 | 精度损失 | 推理加速 | 实现难度 |
|------|--------|----------|----------|----------|
| INT8量化 | 2x | 0.5-2% | 1.5-2x | 低 |
| INT4量化 | 4x | 2-5% | 2-3x | 中 |
| 结构化剪枝 | 1.5-2x | 2-5% | 1.4-1.8x | 中 |
| 知识蒸馏 | 2-5x | 5-10% | 2-5x | 高 |
| LoRA分解 | 1.2-2x | 1-3% | 1.1-1.3x | 低 |

## 组合压缩策略

实际部署中常组合多种技术：

1. **量化 + 剪枝**：
   - 先剪枝移除冗余参数
   - 再量化压缩剩余参数
   - 效果：4-8x压缩，精度损失<5%

2. **蒸馏 + 量化**：
   - 先蒸馏得到小模型
   - 再对小模型量化
   - 效果：8-16x压缩，适合移动端部署

3. **LoRA + 量化**：
   - 量化基座模型
   - LoRA适配器保持高精度
   - 效果：适用于多任务场景

## 面试常考题

### Q1：INT8量化为什么推理速度只提升1.5-2x，而不是理论上的4x（相对FP32）？

**答案要点**：
- 内存带宽瓶颈：虽然计算快了，但数据加载速度受限
- 反量化开销：推理时需将INT8反量化为FP32计算，有转换开销
- 算子融合限制：某些算子无法量化（如LayerNorm、GELU），需要精度转换
- GPU利用率：INT8指令虽快，但可能无法充分利用GPU并行度

### Q2：为什么LLM剪枝通常采用结构化剪枝而非非结构化剪枝？

**答案要点**：
- 硬件友好性：结构化剪枝后的模型是规则的稠密矩阵，可直接用通用硬件（GPU/CPU）加速；非结构化剪枝产生不规则稀疏矩阵，需要专用稀疏矩阵硬件（如NVIDIA Sparse Tensor Cores）
- 实际加速比：非结构化剪枝虽然压缩比更高，但在通用硬件上反而可能变慢（稀疏索引开销）
- 实现复杂度：结构化剪枝只需调整模型维度，代码改动小；非结构化剪枝需要稀疏矩阵库支持

### Q3：知识蒸馏中，为什么Student有时难以超越简单的直接训练小模型？

**答案要点**：
- 容量差异过大：Teacher和Student容量差距太大时，Student难以学习Teacher的复杂决策边界
- 软标签噪声：Teacher的错误预测会误导Student（尤其Teacher在困难样本上也不确定时）
- 训练目标冲突：软标签和硬标签的梯度方向可能冲突，需要仔细调节权重
- 温度系数调优：温度T过大会平滑掉有用信息，过小则退化为硬标签

解决方案：
- 逐步蒸馏：使用中等大小的中间模型作为桥梁
- 数据增强：让Teacher在更多数据上生成软标签
- 自蒸馏：Student也参与训练，不完全依赖Teacher
